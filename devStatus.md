# Vue MDN 개발 현황

작성일: 2026-06-01

## 1. 프로젝트 개요

Vue MDN은 메가스터디 디자인/컴포넌트/프로젝트 산출물을 한곳에서 확인하고, 게시물 형태로 등록/관리하기 위한 내부용 프론트엔드 가이드 및 아카이브 성격의 웹 애플리케이션입니다.

현재 저장소는 `frontend`와 `backend`가 분리되어 있으며, 프론트엔드는 Vue 3 + Vite 기반, 백엔드는 NestJS 기반으로 구성되어 있습니다.

## 2. 현재 기술 구성

### Frontend

- Vue 3.5.x
- Vue Router 4.6.x
- Vite 8.x
- 정적 이미지 에셋 기반 화면 구성
- 브라우저 `localStorage` 기반 임시 게시글 저장

### Backend

- NestJS 11.x
- TypeScript
- Jest 테스트 환경
- 현재는 메모리 배열 기반 게시글 저장 구조

## 3. 구현된 주요 화면

### 공통 레이아웃

- 상단 헤더 `TopHeader`
- 좌측 내비게이션 `LeftNav`
- 라우터 기반 본문 영역 `router-view`
- 404 페이지 `NotFound`

### 소개 영역

- 개요 페이지
- 디자인 원칙 페이지

### 스타일 가이드 영역

- 폰트 스타일 페이지
- 로고, 컬러, 레이아웃, 아이콘 항목은 기본 페이지 또는 미구현 상태로 연결

### 컴포넌트 아카이브 영역

다음 컴포넌트별 페이지 구조가 준비되어 있습니다.

- 배너
- 퀵메뉴
- 게시판
- 탭메뉴
- 리뷰
- 이벤트
- 특강/선생님 탭
- 동영상
- 팝업
- 쿠폰
- 상품 리스트

각 컴포넌트 페이지에는 게시글 등록 화면으로 이동하는 링크가 일부 구현되어 있습니다.

### 프로젝트 아카이브 영역

다음 프로젝트 카테고리 페이지 구조가 준비되어 있습니다.

- 메가선생님
- 메가패스
- 수능/내신
- 대학별고사
- 입시정보
- 이벤트/프로모션
- 온라인서점
- 메가캠페인

일부 프로젝트 페이지는 실제 컴포넌트로 연결되어 있고, 일부는 `DefaultPage`로 임시 연결되어 있습니다.

## 4. 게시글 등록 기능 현황

### 컴포넌트 게시글 등록

`PostEditor.vue`를 통해 컴포넌트 게시글 등록/수정 화면이 구현되어 있습니다.

현재 입력 항목:

- 제목
- 이미지 업로드
- 노출 디바이스 PC/Mobile
- 라벨/태그
- 프로젝트 게시판 링크
- PC 링크
- Mobile 링크
- 담당자
- CSS 코드

현재 저장 방식:

- `localStorage`의 `megaDbPosts` 키에 저장
- 신규 등록 시 `Date.now()` 기반 ID 생성
- 수정 모드는 쿼리스트링의 `id` 값으로 기존 게시글 조회
- 저장 후 `/component/{category}` 경로로 이동

### 프로젝트 게시글 등록

`ProjectPostEditor.vue`를 통해 프로젝트 전용 게시글 등록/수정 화면이 구현되어 있습니다.

현재 입력 항목:

- 프로젝트 제목
- 썸네일 이미지
- 태그
- 프로젝트 게시판 링크
- PC 링크
- Mobile 링크

현재 저장 방식:

- `localStorage`의 `megaDbPosts` 키에 저장
- 신규 등록 시 `Date.now()` 기반 ID 생성
- 수정 모드는 쿼리스트링의 `id` 값으로 기존 게시글 조회
- 저장 후 `/project/{category}` 경로로 이동

## 5. 데이터 및 상태 관리 현황

현재 프론트엔드 게시글 데이터는 서버와 연동되지 않고 브라우저 `localStorage`에 저장됩니다.

공통 유틸리티 `postManager.js`에는 다음 기능이 구현되어 있습니다.

- 카테고리별 게시글 조회
- ID 기반 게시글 삭제

다만 실제 각 목록 페이지에서 `postManager`를 일관되게 사용하는 구조는 아직 정리되지 않았습니다.

## 6. 백엔드 API 현황

`posts` 모듈이 기본 형태로 구현되어 있습니다.

현재 API:

- `POST /posts`: 게시글 등록
- `GET /posts`: 전체 게시글 조회
- `DELETE /posts/:id`: 게시글 삭제

현재 한계:

- 데이터베이스 미연동
- 서버 재시작 시 데이터 소실
- DTO 타입 정의 미완성
- 수정 API 미구현
- 카테고리별 조회 API 미구현
- 이미지 업로드 API 미구현
- 인증/권한 미구현

## 7. 확인된 품질 상태

### 빌드/테스트 확인

- `frontend`: `npm run build` 성공
- `backend`: `npm test -- --runInBand` 성공
- 백엔드 테스트 결과: 3개 테스트 통과

### 개발 환경 경고

PowerShell에서 npm 실행 시 다음 성격의 접근 권한 경고가 출력됩니다.

- `C:\Program Files\nodejs\npm.ps1` 내부 `Test-Path` 접근 권한 경고

명령 자체는 완료되었지만, 개발 환경의 Node/npm 경로 권한 설정은 별도 점검이 필요합니다.

## 8. 주요 리스크 및 미정리 사항

### 라우팅 경로 불일치

일부 메뉴 경로와 라우터 경로가 일치하지 않습니다.

예시:

- 상단 헤더 프로젝트 링크: `/project/mega-teacher`
- 실제 라우터 프로젝트 경로: `/project/megateacher`
- 좌측 메뉴 스타일가이드 경로 일부: `/styleguide/...`
- 실제 라우터 스타일가이드 경로: `/style-guide/...`

### 게시글 등록 링크 오류

일부 컴포넌트 페이지의 등록 링크가 잘못되어 있습니다.

예시:

- `Event.vue`: `../postingpost?category=event`
- 정상 후보: `/posting/post?category=event`

### 데이터 스키마 불일치

컴포넌트/프로젝트 카드와 에디터 사이의 필드명이 완전히 일치하지 않습니다.

예시:

- `ProjectCard`는 `item.tags`, `item.links.board`, `item.links.pc`, `item.links.mo` 구조를 사용
- `ProjectPostEditor`는 `labels`, `MegaProjectLink`, `PcLink`, `MoLink` 구조를 사용

이로 인해 등록한 프로젝트 게시글이 카드에서 의도대로 표시되지 않을 가능성이 있습니다.

### 목록 화면과 등록 데이터 미연동

프로젝트 목록 페이지 일부는 더미 배열 데이터를 직접 가지고 있습니다.

현재 `localStorage`에 저장한 게시글이 모든 목록 화면에 자동 반영되는 구조는 아닙니다.

### 백엔드와 프론트엔드 미연동

백엔드 `posts` API가 존재하지만, 프론트엔드는 아직 해당 API를 사용하지 않고 `localStorage`를 사용합니다.

### 관리자 영역 미구현

상단 헤더에 관리자 메뉴 조건부 노출 구조가 있으나 `/admin` 라우트 및 관리자 기능은 아직 구현되어 있지 않습니다.

## 9. 현재 개발 단계 요약

현재 프로젝트는 다음 상태로 볼 수 있습니다.

- 1차 화면 IA와 주요 메뉴 구조는 구축됨
- 컴포넌트/프로젝트 아카이브의 기본 페이지가 다수 생성됨
- 게시글 등록/수정 UI와 임시 저장 로직이 구현됨
- 백엔드 API의 최소 골격이 생성됨
- 실제 운영 가능한 데이터 관리, API 연동, 관리자 기능, 상세/수정/삭제 플로우는 추가 개발 필요
