# Vue MDN 학습계획

작성일: 2026-06-04

## 1. 학습 목적

이 프로젝트의 가장 중요한 목적은 Vue.js를 기본 개념부터 실전 응용까지 단계적으로 학습하는 것입니다. 단순히 문법을 외우는 것이 아니라, 실제 화면 구조, 라우팅, 컴포넌트 분리, 데이터 흐름, 등록/수정/삭제 기능을 직접 구현하면서 Vue가 실무에서 어떻게 쓰이는지 익히는 데 초점을 둡니다.

최종 목표는 팀원들이 다음 내용을 스스로 설명하고 구현할 수 있는 상태가 되는 것입니다.

- Vue 앱이 어떻게 시작되는지 이해합니다.
- `.vue` 단일 파일 컴포넌트 구조를 이해합니다.
- 컴포넌트를 나누고 조합하는 방식을 익힙니다.
- Vue Router로 페이지를 구성합니다.
- `data`, `computed`, `methods`, `mounted`, `watch`를 실전 흐름에서 사용합니다.
- `v-model`, `v-if`, `v-for`, `:class`, `@click` 같은 템플릿 문법을 실제 UI에 적용합니다.
- props와 emit으로 부모/자식 컴포넌트가 데이터를 주고받는 방식을 이해합니다.
- `localStorage`와 유틸 함수를 사용해 간단한 CRUD 흐름을 만듭니다.
- 같은 기능이 반복될 때 공통 컴포넌트로 분리하는 기준을 익힙니다.

## 2. 프로젝트를 학습 교재로 보는 관점

현재 `vue-mdn` 프로젝트는 Vue 학습에 적합한 구조를 이미 가지고 있습니다.

- `main.js`: Vue 앱 시작점을 학습합니다.
- `App.vue`: 전체 레이아웃의 기준을 학습합니다.
- `router.js`: URL과 화면 컴포넌트 연결 방식을 학습합니다.
- `TopHeader.vue`, `LeftNav.vue`: 공통 UI 컴포넌트 구조를 학습합니다.
- `LeftMenu.js`: 메뉴 데이터를 분리하는 방식을 학습합니다.
- `Banner.vue`: 목록 화면과 게시물 CRUD 흐름을 학습합니다.
- `PostEditor.vue`: 입력 폼, 수정 모드, 이미지 업로드, 저장 로직을 학습합니다.
- `ProjectListPage.vue`: props를 활용한 공통 목록 컴포넌트 구조를 학습합니다.
- `ProjectCard.vue`, `PageComponentListItem.vue`: 자식 컴포넌트와 props/emit 흐름을 학습합니다.
- `postManager.js`: 저장소 접근 로직을 유틸로 분리하는 방식을 학습합니다.

현재 일부 파일은 한글 인코딩이 깨져 있어 학습자가 읽기 어렵습니다. 따라서 학습을 진행하면서 개념 이해와 코드 정리를 함께 진행하는 것이 좋습니다.

## 3. 전체 학습 로드맵

### 1단계: Vue 앱의 시작 구조를 이해합니다.

학습 파일:

- `frontend/src/main.js`
- `frontend/src/App.vue`
- `frontend/vite.config.js`

학습 목표:

- `createApp(App)`의 의미를 이해합니다.
- `app.use(router)`로 플러그인을 등록하는 흐름을 이해합니다.
- `app.mount('#app')`로 Vue 앱이 브라우저에 붙는 방식을 이해합니다.
- Vite alias 설정이 import 경로를 짧게 만들어주는 방식을 이해합니다.
- `App.vue`가 전체 레이아웃의 최상위 컴포넌트라는 점을 이해합니다.

실습 과제:

- `App.vue`에 임시 안내 문구를 추가하고 화면에 표시되는지 확인합니다.
- `main.js`에서 전역 CSS import 순서를 확인합니다.
- `vite.config.js`의 `@components`, `@pages`, `@utils` alias가 실제 import에서 어떻게 쓰이는지 찾아봅니다.

완료 기준:

- 팀원이 Vue 앱이 어디서 시작되고 어디에 마운트되는지 설명할 수 있습니다.
- alias import와 상대경로 import의 차이를 설명할 수 있습니다.

학습 목표:

- 컴포넌트를 import하고 등록해서 사용하는 방식을 이해합니다.
- 공통 UI를 컴포넌트로 분리하는 기준을 이해합니다.

실습 과제:

- `TopHeader.vue`의 메뉴명을 정상 한글로 복구해봅니다.
- `ProjectCard.vue`에 `description` 영역을 추가해봅니다.
- CSS 파일을 import하는 위치와 적용 범위를 확인합니다.

완료 기준:

- 팀원이 컴포넌트 하나가 화면, 로직, 스타일을 어떻게 갖는지 설명할 수 있습니다.
- 공통 헤더나 카드 UI를 왜 컴포넌트로 분리하는지 설명할 수 있습니다.

### 3단계: Vue Router로 페이지를 구성합니다.

- `frontend/src/App.vue`
- `frontend/src/components/TopHeader.vue`

학습 목표:

- `createRouter`, `createWebHistory`를 이해합니다.
- `path`, `name`, `component`, `redirect` 역할을 이해합니다.
- `<router-view />`가 현재 URL에 맞는 화면을 렌더링하는 방식을 이해합니다.
- `<router-link>`와 `this.$router.push()`의 차이를 이해합니다.
- `$route.path`, `$route.query`, `$route.params` 사용법을 이해합니다.

실습 과제:

- `/study/vue-basic` 라우트를 추가하고 임시 학습 페이지를 연결합니다.
- 상단 또는 좌측 메뉴에 새 라우트 링크를 추가합니다.
- 존재하지 않는 URL 접근 시 `NotFound.vue`가 보이는지 확인합니다.

완료 기준:

- 팀원이 URL이 바뀔 때 어떤 컴포넌트가 렌더링되는지 설명할 수 있습니다.
- 라우터 이동 방식인 `router-link`와 `$router.push`를 구분할 수 있습니다.

학습 파일:

- `v-model`로 입력값과 데이터를 연결하는 방식을 이해합니다.
- `v-if`로 조건부 표시를 구현합니다.
- `v-for`로 배열 렌더링을 구현합니다.
- `:class`, `:src`, `:href` 같은 속성 바인딩을 이해합니다.
- `@click`, `@change`, `@keyup.enter` 같은 이벤트 처리를 이해합니다.

실습 과제:

- 빈 목록일 때 표시되는 문구를 정상 한글로 수정합니다.

- 배열 데이터를 화면에 반복 출력하는 방법을 구현할 수 있습니다.

### 5단계: Options API 핵심 옵션을 이해합니다.

학습 파일:

- `frontend/src/pages/posting/PostEditor.vue`
- `frontend/src/components/LeftNav.vue`
- `frontend/src/components/ProjectListPage.vue`

학습 목표:

- `data`는 컴포넌트 내부 상태라는 점을 이해합니다.
- `computed`는 기존 값을 바탕으로 계산되는 값이라는 점을 이해합니다.
- `mounted`는 화면이 처음 렌더링된 뒤 실행되는 초기화 작업이라는 점을 이해합니다.
- `methods`는 사용자의 행동에 반응하는 함수라는 점을 이해합니다.
- `watch`는 값 변화를 감지해 추가 작업을 실행한다는 점을 이해합니다.

실습 과제:

- `PostEditor.vue`의 `isEditMode`가 어떤 URL에서 true가 되는지 확인합니다.
- `ProjectListPage.vue`에서 `category` prop이 바뀌면 목록을 다시 불러오는 흐름을 확인합니다.
- `LeftNav.vue`에서 라우트 변경 시 메뉴가 열리는 흐름을 정리합니다.

완료 기준:

- 팀원이 각 옵션의 역할을 실제 파일 예시로 설명할 수 있습니다.
- `computed`와 `methods`를 언제 구분해서 쓰는지 말할 수 있습니다.

## 6. 첫 번째 실습 과제

첫 번째 실습은 “깨진 한글 문구를 정상화하면서 Vue 구조 읽기”로 진행합니다.

대상 파일:

- `frontend/src/components/TopHeader.vue`
- `frontend/src/data/LeftMenu.js`
- `frontend/src/pages/component/Banner.vue`

작업 목표:

- 화면에 보이는 메뉴명과 버튼명을 정상 한글로 수정합니다.
- 라우터 이동이 깨지지 않는지 확인합니다.
- `v-for`, `router-link`, `:class`가 각각 어디에 쓰였는지 주석으로 정리합니다.

이 과제는 단순 수정처럼 보이지만, 팀원이 Vue의 기본 렌더링 구조와 라우터 구조를 가장 부담 없이 읽기 좋은 출발점이 됩니다.
